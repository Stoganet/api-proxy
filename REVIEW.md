# Review instructions

## What Important means here

Reserve 🔴 Important for findings that would break behavior, leak data, or create a security
vulnerability: incorrect logic, auth bypasses, unhandled error paths that silently swallow
failures, SQL injection, JWT signing method not validated in the keyfunc, and PII in logs.
Style, naming, and refactoring suggestions are 🟡 Nit at most.

## Import cycle — always check, always Important

`internal/auth` must never import `internal/clients/jellyfin`. The auth package owns its
own `JellyfinAuthenticator` interface and result types to keep the packages decoupled. Flag
any import of the jellyfin package inside `internal/auth/` as 🔴 Important.

## Cap the nits

Report at most five 🟡 Nits per review. If you found more, say "plus N similar items" in the
summary. If all findings are nits, lead the summary with "No blocking issues."

## Do not report

- Formatting, import ordering, or lint issues — golangci-lint handles these in CI
- Anything inside `internal/gen/` — generated by oapi-codegen, never edited by hand. Skip these hunks entirely when reading `gh pr diff` output.
- The `_ = tx.Rollback()` deferred rollback pattern — this is intentional

## Always check

- JWT keyfuncs validate the signing method before returning the key
- All SQL queries use parameterised placeholders, never string concatenation
- New API routes have a corresponding handler test
